Timer

The Timer Object

Timer → Object

An object containing various methods related to timing of events and processes.

Not to be confused for a more robust date and time system - everything in this object is to be used "relatively." 

Timer now → Double

Returns the current UNIX timestamp in seconds.

Example:

> Timer now 
1.297049761144438e9

Timer sleep: duration → @ok
  | duration is-a?: Number

Delays the current process's execution by duration microseconds.

Example:

> Timer sleep: 1 millisecond
@ok

Timer do: action every: duration → Process
  | action is-a?: Block
  | duration is-a?: Number

Spawns a new process that repeatedly calls action at intervals of duration microseconds.

Example:

> p = Timer do: { "hello" print } every: 0.125 seconds
<process 47>
> Timer sleep: 0.25 seconds
@ok
> p stop
@ok

Timer do: action after: duration → Process
  | action is-a?: Block
  | duration is-a?: Number

Spawns a new process that calls action after duration microseconds.

Example:

> p = Timer do: { "hello" print } after: 0.125 seconds
<process 50>
> Timer sleep: 0.25 seconds
@ok
> p stop
@ok

action time → Double
  | action is-a?: Block

Times how long it takes for action call to complete, returning the duration in seconds.

Example:

> { Timer sleep: 0.1 seconds } time
0.1001131534576416

Unit Helpers

There are many methods defined for use with Timer so you don't have to do a bunch of calculations or repeat a bunch of 0s. 

n us → Number
  | n is-a?: Number

Yields n microseconds. This is effectively a no-op, returning the same number back, since Timer already uses microsecond precision.
